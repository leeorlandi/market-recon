# Procurement vs. Compete Decision Framework

Used by Market Recon (`/market-recon`) to flag pay-to-play opportunities in the digest.

---

## When to flag a player as Pay-to-Play

Score each of the following. 3+ = flag as procurement target.

| Signal | Yes / No |
|---|---|
| Player has a network effect moat (data, integrations, community) | |
| Player is already embedded in your target buyer's stack | |
| Building equivalent capability would take 12+ months | |
| Their data/coverage is not replicable from public sources | |
| They have a partner program or reseller structure | |
| Cost of partnership < cost of competitive displacement | |

---

## Engagement Tiers

### Tier A — Call them now
- Immediate capability gap
- They have something you need in the next 90 days
- Partner or procurement motion exists

### Tier B — Watch and evaluate
- Useful but not urgent
- Monitor for pricing changes, new offerings, or strategic pivots

### Tier C — Note and move on
- Not a near-term fit, but relevant to the landscape

---

## What to include in the digest entry

For each pay-to-play target, note:
- What specific capability to procure
- Likely contact point (partnerships team, sales, API/marketplace)
- Estimated commercial model (API usage, seats, rev share)
- Any known existing relationships or integrations
